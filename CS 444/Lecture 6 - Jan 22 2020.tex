\subsubsection{LR(0) Parsing}

\begin{ex}-\\
\begin{itemize}
\item \(S \rightarrow E \$\)
\item \(E \rightarrow E + a \)
\item \(E \rightarrow a \) 
\end{itemize}
You can not parse this grammar with LL because the praser does not how many times to apply the recursive rule.  
\end{ex}

\subsubsection{LR(0) Parsing Algorithm}
\begin{itemize}
\item for each a in \(\ell \$\) (\(\ell\) is input)
\begin{itemize}
\item While ( \(Reduce(\text{stack}) = \{ A \rightarrow \gamma\}\))
\begin{itemize}
\item Pop \(|\gamma|\) times
\item Push A
\end{itemize}
\item if (\(Reject(stack + a)\)) then throw ERROR
\item push a
\end{itemize}
\item \(Reject(\alpha) = \alpha\) is not a viable prefix
\item \(Reduce(\alpha) = \{A \rightarrow \gamma | \exists P, \alpha = \beta \gamma \) and \( \beta A \) is a viable prefix \(\}\)
\end{itemize}

\begin{definition}
\(\alpha\) is a \underline{viable prefix} if it is a prefix of a some sentential form which is any sequence that can be derived from the start symbol.
\end{definition}

\begin{itemize}
\iten Only some viable prefixes ever occur as stacks and these subsets form a regular language.
\end{itemize}
