\section{Introduction to Computer Security and Privacy}
\begin{itemize}
\item The goal of computer security is to be able to identify security and privacy issues in various aspects of computing. (Programs, operating systems, Networks, Internet applications, Databases)
\item The Secondary goal is to be able to use this ability to design systems that are more protective of security and privacy
\item In the context of computers, security generally means three things : 
\begin{itemize}
\item Confidentiality
\item Integrity
\item Availability
\end{itemize}
\item If CIA is satisfied, that a system is said to be secure
\item \textbf{Privacy} is "Informational self-determination", which means you get to control information about you
\end{itemize}

\subsection{Terminology}
\begin{itemize}
\item \textbf{Assets} - Things we might want to protect
\item \textbf{Vulnerabilities} - Weaknesses in a system that may be \textbf{exploited} in order to cause loss or harm
\item \textbf{Threats} - A loss or harm that might befall a system
\item \textbf{Threat Model} - A set of threats we are undertaking to defend against
\item \textbf{Attack} - An action which exploits a vulnerability to execute a threat
\item \textbf{Control/Defence} - Removing or reducing a vulnerability
\end{itemize}

\subsection{Defence Steps}
\begin{itemize}
\item \textbf{Prevent it} 
\item \textbf{Deter it} - Make the attack harder or more expensive
\item \textbf{Deflect it} - Make your self less attractive to attackers
\item \textbf{Detect it} - Notice that attack is occurring or occurred. 
\item \textbf{Recover from it} : Mitigate the effects of the attack
\end{itemize}

Its also worth noting the \textbf{Principle of Easiest Penetration} states a system is only as strong as its weakest link.

\subsection{Methods of Defence}

\begin{itemize}
\item Cryptography
\item Software controls
\item Hardware controls
\item Physical controls
\item Policies and procedures
\end{itemize}
